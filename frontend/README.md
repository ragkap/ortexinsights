# Ortex Insights Frontend

A Next.js frontend for browsing and filtering Ortex market intelligence insights.

## Setup

1. Install dependencies:
   ```bash
   cd frontend
   npm install
   ```

2. Ensure the backend has generated `insights.json` in the parent directory:
   ```bash
   cd ..
   python main.py  # Generates insights.json
   cd frontend
   ```

3. Start the development server:
   ```bash
   npm run dev
   ```

4. Open [http://localhost:3000](http://localhost:3000) in your browser

## Features

- **Responsive Design** — Works on desktop, tablet, and mobile
- **Real-time Filtering** — Filter by content type, ticker, severity, theme, topics, and exchange
- **Insight Cards** — Displays headline, body preview, severity badge, and content type
- **Detail View** — Expandable details for each insight
- **Pagination** — Configurable results per page

## Filters

- **Content Type** — pulse, earnings_preview, trader_note, weekly_digest, convergence_report, market_commentary
- **Ticker** — Search by stock ticker (e.g., NYSE:AAPL, AAPL)
- **Exchange** — Filter by exchange (NYSE, NASDAQ, LSE, etc.)
- **Severity** — critical, high, medium, low
- **Theme** — week_ahead, overall, short_interest, analyst, options, convergence, general, events, insider, money_flow
- **Topics** — Comma-separated list (e.g., short_interest,options)
- **Results per Page** — 10, 25, 50, or 100

## Data Source

The frontend reads from `../insights.json` generated by the backend Python script. The API route (`/api/insights`) filters and serves the data.

## Build for Production

```bash
npm run build
npm start
```

## Troubleshooting

- **"No insights found"** — Run `python main.py` in the parent directory to generate `insights.json`
- **Port 3000 in use** — Run `npm run dev -- -p 3001` to use a different port
- **Module not found** — Run `npm install` again to ensure all dependencies are installed
